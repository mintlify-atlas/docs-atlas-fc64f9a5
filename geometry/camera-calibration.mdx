Camera calibration is the process of determining the parameters that relate 3D world coordinates to 2D image coordinates. These parameters fall into two groups: **intrinsic** parameters (internal optics) and **extrinsic** parameters (pose in the world).

## The camera model

A pinhole camera maps a 3D world point $\mathbf{M} = (X, Y, Z, 1)^\top$ to a 2D image point $\mathbf{m} = (u, v, 1)^\top$ via:

$$
\mathbf{m} \sim P\,\mathbf{M}, \qquad P = K\,\bigl[\,R\;|\;\mathbf{t}\,\bigr]
$$

### Intrinsic matrix $K$

$$
K = \begin{pmatrix}f_x & s & c_x \\ 0 & f_y & c_y \\ 0 & 0 & 1\end{pmatrix}
$$

| Parameter | Meaning |
|---|---|
| $f_x, f_y$ | Focal lengths in pixel units (horizontal and vertical) |
| $c_x, c_y$ | Principal point (optical axis in image coordinates) |
| $s$ | Skew coefficient (nearly zero for modern cameras) |

### Extrinsic parameters: $R$ and $\mathbf{t}$

$R \in SO(3)$ is the rotation matrix and $\mathbf{t} \in \mathbb{R}^3$ the translation that express the world frame in the camera frame. Together they form the $3\times 4$ matrix $[R\,|\,\mathbf{t}]$.

The full $3\times 4$ projection matrix $P$ therefore has 11 degrees of freedom (up to scale).

## Linear calibration via DLT

The **Direct Linear Transform (DLT)** estimates $P$ from a set of known 3D–2D correspondences $\{\mathbf{M}_i \leftrightarrow \mathbf{m}_i\}$.

Each correspondence yields two linear equations in the 12 entries of $P$. Stacking $n \geq 6$ correspondences gives the system:

$$
A\,\mathbf{p} = \mathbf{0}, \qquad \mathbf{p} = \text{vec}(P)
$$

The solution is the **right singular vector** of $A$ corresponding to the smallest singular value — computed via SVD.

<Note>
Numerical stability requires normalising the input coordinates before applying DLT (translate centroid to origin, scale so mean distance to origin is $\sqrt{2}$).
</Note>

### Least-squares and SVD

For overdetermined systems $A\mathbf{x} = \mathbf{b}$ the least-squares solution minimises $\|A\mathbf{x} - \mathbf{b}\|^2$. The SVD decomposition $A = U\Sigma V^\top$ gives the solution directly: $\mathbf{x} = V\boldsymbol{\Sigma}^{-1}U^\top\mathbf{b}$. For the homogeneous case ($\mathbf{b} = \mathbf{0}$) the solution is the last column of $V$.

Recovering $K$, $R$, $\mathbf{t}$ from $P$ uses **RQ decomposition** (a variant of QR decomposition).

## Nonlinear optimisation and lens distortion

Real lenses introduce **radial distortion**: straight lines appear curved. The distorted image coordinates $(u_d, v_d)$ relate to the undistorted $(u, v)$ by:

$$
u_d = u\,(1 + k_1 r^2 + k_2 r^4 + \cdots), \quad r^2 = u^2 + v^2
$$

where $k_1, k_2, \ldots$ are radial distortion coefficients. After the linear DLT initialisation, a **nonlinear least-squares** optimiser (e.g., Levenberg–Marquardt) jointly refines all parameters including distortion.

## RANSAC for robust calibration

When correspondences include outliers, ordinary least-squares gives poor estimates. **RANSAC (RANdom SAmple Consensus)** iteratively:

<Steps>
  <Step title="Sample a minimal set">
    Randomly pick the minimum number of points needed to fit the model (e.g., 6 for DLT).
  </Step>
  <Step title="Fit the model">
    Compute the candidate parameter vector from the minimal sample.
  </Step>
  <Step title="Count inliers">
    Count how many of the remaining points agree with the model within a threshold $\epsilon$.
  </Step>
  <Step title="Keep the best model">
    Retain the hypothesis with the most inliers. After convergence, refit using all inliers.
  </Step>
</Steps>

```matlab Camera calibration with checkerboard (CV02_Calibration.m)
% Auto-generated by MATLAB cameraCalibrator app

% Define calibration images
imageFileNames = {
    '../images/calib_example/Image1.tif', ...
    '../images/calib_example/Image2.tif', ...
    % ... (15 images total)
    '../images/calib_example/Image15.tif', ...
};

% Detect checkerboard corners
[imagePoints, boardSize, imagesUsed] = detectCheckerboardPoints(imageFileNames);
imageFileNames = imageFileNames(imagesUsed);

% Generate 3D world coordinates (square size = 25 mm)
squareSize  = 25;
worldPoints = generateCheckerboardPoints(boardSize, squareSize);

% Estimate camera parameters (2 radial distortion coefficients)
[cameraParams, imagesUsed, estimationErrors] = estimateCameraParameters( ...
    imagePoints, worldPoints, ...
    'EstimateSkew', false, ...
    'EstimateTangentialDistortion', false, ...
    'NumRadialDistortionCoefficients', 2, ...
    'WorldUnits', 'mm');

% Visualise reprojection errors and extrinsics
figure; showReprojectionErrors(cameraParams, 'BarGraph');
figure; showExtrinsics(cameraParams, 'CameraCentric');
displayErrors(estimationErrors, cameraParams);

% Undistort an image using the calibrated parameters
originalImage   = imread(imageFileNames{1});
undistortedImage = undistortImage(originalImage, cameraParams);
```

```matlab RANSAC line fitting (CV02_Ransac.m)
% Robust line fitting with RANSAC
clt
N = 100;
x = (1:N)';
m = 4;  b = -2;
y = m*x + b + 25*(rand(N,1)-0.5);  % noisy line

% Inject outliers
y(10)=490; y(20)=20; y(50)=90; y(75)=N;
y(15)=440; y(25)=120; y(55)=70; y(85)=N/2;

mr=0; br=0; yr=mr*x+br; our=Inf;

for k = 1:1000
    % Pick two random points
    k1 = fix(N*rand)+1;
    k2 = k1;
    while (k2==k1), k2 = fix(N*rand)+1; end

    % Fit a line through the sample
    ms = (y(k2)-y(k1))/(x(k2)-x(k1));
    bs = y(k2) - ms*x(k2);
    ys = ms*x + bs;

    % Count outliers (threshold = 15 pixels)
    e  = abs(y - ys);
    o  = sum(e > 15);

    % Keep best model so far
    if o < our
        mr=ms; br=bs; yr=ys; our=o;
        fprintf('m=%6.2f  b=%6.2f  outliers=%d\n', mr, br, our);
    end
end
```

## Calibration workflow summary

<AccordionGroup>
  <Accordion title="How many images are needed for calibration?">
    In practice 10–20 images of a calibration pattern (checkerboard) taken from different angles and distances give reliable results. The MATLAB `cameraCalibrator` app and OpenCV's calibration routine both accept multiple views.
  </Accordion>
  <Accordion title="What is the reprojection error?">
    After calibration the reprojection error is the average Euclidean distance between the observed image points and the points predicted by projecting the 3D world points through the estimated $P$. A reprojection error below 1 pixel is considered good.
  </Accordion>
  <Accordion title="When is lens distortion important?">
    Wide-angle and fisheye lenses introduce significant radial distortion. For standard lenses in robotics or structure-from-motion, even small distortions compound over large scenes. Always estimate and correct distortion if metric accuracy matters.
  </Accordion>
  <Accordion title="How does RANSAC improve calibration?">
    Mislabelled or partially occluded corners act as outliers. RANSAC finds the largest set of consistent correspondences before refitting, giving an estimate close to the true parameters even when 30–50 % of matches are corrupted.
  </Accordion>
</AccordionGroup>
